StandardScout

RAG-Based Building Code Assistant

StandardScout is an online building-code copilot powered by LLMs. It combines a local RAG knowledge base for ADA, ABA, and GSA P-100 with optional PDF upload and web search, so users can turn sources on or off as needed.

Live demo Back to Projects

Two interaction modes (Text and Conversational) share the same reasoning pipeline — the difference is UI-only. Each answer also generates a minimal “blueprint-style” technical diagram to make requirements easier to verify.

What it solves

Building-code documents are long, cross-referenced, and often require jumping between sections to confirm a single dimension or exception. StandardScout is designed to speed up exploration without “making up” rules: it responds only when the requested value is explicitly present in the retrieved sources and otherwise reports “Not found in retrieved sources.”

For quick verification

Ask a targeted question, get a one-paragraph answer, and see exactly which sources were used.

For learning (not overreliance)

A friendly, citation-first style encourages users to confirm requirements in the original documents.

For real projects

Turn local RAG, uploads, and web search on/off depending on what a project needs.

Core features

Selectable sources per question

Toggle any combination of: local RAG (ADA/ABA/GSA P-100), uploaded PDFs (session indexing), and web search. Results are merged, deduplicated, and reranked before answer generation.

Local • Upload • Web

Dual interaction modes

Text-based mode for direct Q&A and a Conversational mode for a more dialogue-like flow. Both modes use the same retrieval + grounding logic.

Text • Conversational

Citation-backed output

Answers are formatted with explicit references to the retrieved chunks so it’s easy to trace the basis for each claim.

Grounded • Traceable

Automatic technical diagram

Every response also produces a minimal technical diagram (plan/section/elevation) that includes only dimensions explicitly stated in the answer.

Diagram • Blueprint

Interface

The current prototype uses a Gradio UI in Colab, with a clean “mode + sources + question” layout. Upload indexing is optional and stores a vector-store handle per session for retrieval.

How the pipeline works

A lightweight “orchestra” orchestrator routes a single question through a consistent set of steps:

  1. Rewrite the question (optional) to improve retrieval.
  2. Retrieve from enabled layers (local / upload / web).
  3. Deduplicate + normalize chunks.
  4. Rerank with an LLM and keep top results.
  5. Generate a single-paragraph answer grounded only in retrieved chunks.
  6. Format references so users can trace the basis for each statement.
  7. Generate an image prompt and render a minimal technical diagram (square, white background, line art).

The grounding rule is strict: if a dimension/threshold/exception is not explicitly present in retrieved chunks, the assistant reports it as not found rather than guessing.

Tech stack

Retrieval

Chroma (local), OpenAI Vector Store for uploads, and Google Custom Search for web snippets.

Reasoning

Orchestrator with optional query rewrite, LLM reranking, and a strict “no invention” grounding rule.

UI

Gradio interface (prototype) — designed to be embedded inside this project page when hosted.

Visualization

Auto-generated minimal technical diagrams from each Q&A (prompted from the grounded answer).

Status

Current version: functional prototype (local RAG + optional upload + optional web search), with citation-backed answers and per-answer diagram generation. Early evaluations were promising.
